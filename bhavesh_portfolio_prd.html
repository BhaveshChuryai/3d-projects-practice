02 — ARCHITECTURE

The Right Way to Build This

🏗️Layer Model (inspired by PeachWeb)
Layer 1 (fixed, z=0): Three.js WebGL canvas — fullscreen, fixed position, always behind everything. Contains all 3D: particles, floating geometry, light effects, animated 3D subject.

Layer 2 (scroll, z=10): HTML content sections — fully transparent background, scroll normally over the canvas. Each section has a dark glass overlay so text is readable against the 3D background.

Layer 3 (fixed, z=100): Nav, cursor, chatbot, music button. Always on top.

Key insight: PeachWeb never puts a 3D model inside a div container. The 3D is the page. HTML floats on top.
📐Scroll Architecture
Use window.scrollY mapped to a 0→1 progress value. Drive ALL animations from this single value — camera position, object rotation, particle density, section reveal opacity. This is exactly how PeachWeb's scroll works. No GSAP ScrollTrigger needed, just requestAnimationFrame + scroll listener.

05 — ANIMATION PRINCIPLES

What Makes PeachWeb Feel Different

01
Scroll = Camera Movement, Not Page Jump

Scrolling should feel like flying through a world, not clicking through pages. Map scrollY to camera.position.z so the 3D world responds to every pixel of scroll. The content sections appear to "grow" out of the 3D environment.

02
One Animation Loop, Everything Driven From It

Single requestAnimationFrame loop handles: camera, particles, geometry rotation, section reveals, cursor parallax. No multiple setIntervals fighting each other. This is why the previous version was buggy — too many competing animation sources.

03
Mouse Parallax = Depth, Not Distraction

Mouse moves camera position by ±2 units smoothly (lerp, not snap). Makes the 3D world feel alive. DON'T rotate Iron Man or subject 90 degrees — subtle 5–10 degree range only. Use easing: camera.position.x += (targetX - camera.position.x) * 0.05

04
Entrance Animations: Stagger + Easing

Never animate all elements simultaneously. Stagger by 100–150ms. Use CSS cubic-bezier(0.22, 1, 0.36, 1) — the "overshoot spring" feel. Elements should slide up AND fade in together. First text appears 300ms after load, each next element 150ms later.

05
60 FPS is Non-Negotiable

Limit particle count to 1500–2000. Use BufferGeometry (not Geometry). Set renderer.setPixelRatio(Math.min(devicePixelRatio, 2)). Avoid per-frame DOM reads (getBoundingClientRect in rAF loop = janky). Cache element positions on resize only.

07 — RULES

Hard Rules to Never Break

🚫
No position:absolute on content over canvas

Creates z-index wars and layout chaos. Use normal document flow for sections.

🚫
No multiple canvases fighting

One fixed fullscreen canvas for the hero universe. One small canvas for the skills section only.

🚫
No Iron Man image on canvas

PNG images inside Three.js scenes don't work well without loaders. Use CSS for images.

🚫
No inline styles fighting CSS classes

All styles in one <style> block. Never mix inline style="" with class-based styles on same element.

✅
Sections have dark glass overlay

Each section: background: rgba(7,0,15,0.85); backdrop-filter: blur(2px); so 3D shows through but text is readable.

✅
One source of truth for scroll progress

let scrollProgress = window.scrollY / (document.body.scrollHeight - window.innerHeight); Use this everywhere.
